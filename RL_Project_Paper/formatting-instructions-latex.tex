\begin{abstract}
\begin{quote}
We investigate how different state representations interact with function approximation methods in reinforcement learning, using the game Snake as a testbed. We compare three function approximation approaches — hand-crafted linear features, hash-based tile coding, and a Double DQN neural network with experience replay — across three state representations of increasing complexity (11, 109, and 126 dimensions), each trained for 20,000 episodes across 5 random seeds. Our experiments show that the neural network achieves the strongest and most consistent performance across all representations, reaching a mean score of 29.05 on extended features and exceeding the hand-coded greedy heuristic (22.89). Tile coding achieves 22.69 on compact features but collapses with dimensionality (3.21 on local, 1.32 on extended) due to hash collisions. Linear function approximation learns danger avoidance but not food-seeking, scoring below 4.0 across all representations. These results demonstrate that function approximation method and representation complexity interact strongly: tile coding is an efficient middle ground for well-designed low-dimensional features, while neural networks generalize robustly across representations when stabilized by Double DQN and experience replay.
\end{quote}
\end{abstract}

\section{Introduction}

Reinforcement learning (RL) has achieved remarkable success in game-playing domains, from TD-Gammon's near-expert backgammon play \cite{tesauro1995tdgammon} to DQN's human-level Atari performance \cite{mnih2015dqn}. A recurring theme in this progression is the interaction between state representation and function approximation: the choice of how to represent the state, and how to approximate the value function over that representation, jointly determine whether an agent can learn effectively.

This interaction is well-studied at the extremes — tabular methods on small state spaces and deep neural networks on raw pixel inputs — but the middle ground is less explored. When is a simple linear model sufficient? When does automatic feature construction (tile coding) outperform hand-crafted features? At what point does nonlinear approximation become necessary? These questions have practical implications for domains where deep RL is overkill but tabular methods are insufficient.

We study these questions using the classic game Snake as a testbed. Snake is well-suited for this investigation because it offers a natural spectrum of state representations: from a minimal 11-dimensional binary feature vector (danger signals and food direction) to a rich 126-dimensional vector combining a local grid window with continuous distance features. The game's difficulty also increases within an episode as the snake grows, creating a non-stationary effective state space that challenges all methods.

Our contributions are as follows. First, we present a systematic comparison of three function approximation methods — hand-crafted linear features, hash-based tile coding, and a Double DQN neural network with experience replay — across three state representations of increasing complexity. Second, we demonstrate that tile coding dramatically outperforms hand-crafted linear features on the compact representation ($22.69$ vs.\ $0.80$), highlighting the value of automatic feature construction. Third, we show that Double DQN with experience replay achieves the highest overall score ($29.05$ on extended features), exceeding the greedy heuristic by 27\% and maintaining low variance across all three representations. Fourth, we provide evidence that the choice of function approximation method interacts strongly with representation complexity: tile coding degrades monotonically with dimensionality while neural networks do not.

\section{Background}

\subsection{Temporal-Difference Learning}

Temporal-difference (TD) learning \cite{sutton1988td} updates value estimates using the difference between successive predictions, without waiting for episode completion. For control, SARSA \cite{rummery1994sarsa} updates the action-value function using the transition $(S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1})$:
\begin{multline}
    Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \\
    \alpha \left[ R_{t+1} + \gamma Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t) \right]
\end{multline}
where $\alpha$ is the learning rate and $\gamma$ is the discount factor. SARSA is on-policy: it evaluates and improves the policy currently being followed, including its exploratory actions.

\subsection{Function Approximation}

When the state space is too large for tabular methods, the value function must be approximated. In \textbf{linear function approximation}, the action-value function is:
\begin{equation}
    \hat{q}(s, a; \mathbf{w}) = \mathbf{w}^\top \mathbf{x}(s, a)
\end{equation}
where $\mathbf{x}(s, a) \in \mathbb{R}^d$ is a feature vector and $\mathbf{w} \in \mathbb{R}^d$ is a weight vector. The semi-gradient SARSA update becomes:
\begin{equation}
    \mathbf{w} \leftarrow \mathbf{w} + \alpha \delta_t \, \mathbf{x}(S_t, A_t)
    \label{eq:linear_update}
\end{equation}
where $\delta_t = R_{t+1} + \gamma \hat{q}(S_{t+1}, A_{t+1}; \mathbf{w}) - \hat{q}(S_t, A_t; \mathbf{w})$.

\textbf{Tile coding} \cite{sutton2018reinforcement,sherstov2005tilecoding} automatically constructs binary feature vectors. Multiple overlapping tilings partition the normalized state space, and the feature vector has one active bit per tiling. The value function is the sum of weights at active tile indices. Nearby states share most active tiles, providing smooth generalization.

\textbf{Neural network approximation} replaces the linear model with a nonlinear function parameterized by $\boldsymbol{\theta}$, updated via backpropagation. \textbf{Double DQN} \cite{vanhasselt2016deep} addresses the overestimation bias of vanilla DQN by decoupling action selection from evaluation: the online network selects the greedy next action $\hat{a} = \arg\max_a Q(S_{t+1}, a; \boldsymbol{\theta})$, while a periodically-synchronized target network scores it:
\begin{equation}
    Y_t = R_{t+1} + \gamma Q(S_{t+1}, \hat{a}; \boldsymbol{\theta}^-)
\end{equation}
Combined with a replay buffer that breaks temporal correlations in the training data, this approach stabilizes learning with nonlinear function approximation.

\subsection{The Deadly Triad}

Sutton and Barto \shortcite{sutton2018reinforcement} identify the \emph{deadly triad}: the combination of function approximation, bootstrapping, and off-policy learning can cause divergence. Tsitsiklis and Van Roy \shortcite{tsitsiklis1997analysis} proved that on-policy TD with linear function approximation converges with probability 1, while Baird \shortcite{baird1995residual} showed that off-policy TD with linear FA can diverge even in simple cases. Van Hasselt et al.\ \shortcite{vanhasselt2018deadly} confirmed these risks empirically in deep RL settings.

Our linear and tile coding agents use on-policy semi-gradient SARSA, maintaining the full convergence guarantee. Our neural network agent departs from on-policy sampling via experience replay, entering the off-policy leg of the deadly triad; we mitigate the resulting instability with Double DQN (reduces overestimation bias) and a hard-synchronized target network (stabilizes regression targets), following established practice in deep RL \cite{mnih2015dqn,vanhasselt2016deep}.

\section{Related Work}

\subsection{Snake as an RL Domain}

Snake has been studied with a range of RL methods. Wei et al.\ \shortcite{wei2018autonomous} applied DQN with CNN processing of raw pixel frames, introducing dual experience replay and a training gap strategy. Bonnici et al.\ \shortcite{bonnici2022exploring} compared Q-learning, SARSA, PPO, and A* across vector, CNN, and raycasting representations, finding PPO with raycasting performed best. Sebastianelli et al.\ \shortcite{sebastianelli2021deep} showed that a sensor-based 11-value state representation trains far more easily than CNN approaches. A Stanford CS230 project by Jiang and Mai compared an 11-boolean feature vector against grayscale screenshots, finding the compact features achieved sevenfold higher scores \cite{jiang2021snakeai}. Ma et al.\ \shortcite{ma2016snake} found that SARSA outperformed Q-learning on Snake with 128 discrete states and proved that optimal Snake play involves NP-hard subproblems.

\subsection{Linear FA vs.\ Neural Networks}

Liang et al.\ \shortcite{liang2016shallow} demonstrated that hand-crafted features with linear function approximation can match DQN performance on Atari games. Elfwing et al.\ \shortcite{elfwing2018sigmoid} compared TD($\lambda$) with neural networks versus linear FA in Tetris, finding that neural networks significantly outperformed linear methods when the value function was inherently nonlinear. Tesauro \shortcite{tesauro1995tdgammon} showed that adding hand-crafted expert features to a neural network dramatically improved TD-Gammon's performance.

\subsection{Positioning}

No prior work has systematically compared function approximation methods — specifically hand-crafted linear features, tile coding, and a neural network — across multiple state representations for Snake using a common experimental framework. Most existing studies use off-policy DQN and focus on the neural network side. We fill this gap by holding the training framework fixed and varying both the approximation method and the representation.

\section{Methods}

\subsection{Environment}

We implement a custom Snake environment on a $20 \times 20$ grid following the Gymnasium interface \cite{gymnasium}. The snake starts at the center with length 3, facing right. The action space is $\mathcal{A} = \{\text{STRAIGHT}, \text{LEFT}, \text{RIGHT}\}$ relative to the current heading, eliminating the possibility of reversing into the snake's body. Rewards are $+10$ for eating food, $-10$ for dying (wall or self-collision), and $-0.1$ per step to encourage efficiency. The discount factor is $\gamma = 0.95$. Episodes terminate on death or after $3 \times \text{grid\_size}^2 = 1{,}200$ steps (truncation).

For the neural network agent, we additionally apply potential-based reward shaping \cite{ng1999reward}: a bonus of $F(s, s') = 0.5 \times (d(s) - d(s'))$ is added each step, where $d$ is the Manhattan distance from the snake's head to the food. This bonus is omitted when food is eaten, as the food position resets. Potential-based shaping does not alter the optimal policy \cite{ng1999reward} but provides a denser reward signal that speeds convergence.

\subsection{State Representations}

We compare three representations of increasing complexity.

\textbf{Compact (11 dimensions):} Three binary danger signals (wall or body one step ahead in each relative direction), four one-hot direction bits, and four binary food-direction indicators. This matches the standard design used in most Snake RL implementations \cite{loeber2020snakeai,zhou2023snake}.

\textbf{Local Neighborhood (109 dimensions):} A $5 \times 5$ grid window centered on the snake's head, with each cell one-hot encoded as [empty, food, wall, body] (100 features), plus food direction (4) and snake length bucket (5).

\textbf{Extended (126 dimensions):} The local neighborhood features augmented with normalized Manhattan distance to food (1), distance to the nearest body segment in each cardinal direction (4), distance to walls (4), normalized snake length (1), current direction one-hot (4), and explicit danger signals (3).

For the linear and tile coding models, state-action features are constructed by one-hot action stacking: the state vector is placed in the block corresponding to the action, with zeros elsewhere, yielding feature dimensions of 33, 327, and 378 respectively.

\subsection{Algorithms}

All three algorithms use $\varepsilon$-greedy exploration with $\varepsilon$ decaying linearly from 1.0 to 0.01 over 80\% of the 20,000 training episodes.

\textbf{Linear FA SARSA.} The update follows Equation~\ref{eq:linear_update} with learning rate $\alpha = 0.01$.

\textbf{Tile Coding SARSA.} We use overlapping tilings with 4 tiles per dimension and a hash table of 262,144 entries. We use 8 tilings for the compact representation and 4 tilings for local and extended representations (where additional tilings provide diminishing returns given the fixed hash table size). The effective learning rate is $\alpha = 0.05 / n_{\text{tilings}}$ per the standard convention \cite{sutton2018reinforcement}. Feature normalization ranges are estimated by sampling 1,000 random episodes before training.

\textbf{Double DQN.} A two-layer network with hidden dimensions (256, 128) and ReLU activations outputs Q-values for all three actions simultaneously. We use a replay buffer of capacity 50,000 transitions, sampling mini-batches of 64 transitions per gradient step. A target network is hard-copied from the online network every 100 episodes. The training target uses Double DQN as described in the Background section: the online network selects the greedy next action and the target network evaluates it, reducing overestimation bias. The loss is Huber (smooth $\ell_1$) for robustness to large TD errors. We use the Adam optimizer \cite{kingma2015adam} with learning rate $10^{-3}$ and gradient clipping at max norm 5.0. Distance-based reward shaping is applied during training.

\subsection{Baselines}

We compare against two baselines: a \textbf{random policy} (uniform random actions) and a \textbf{greedy heuristic} that avoids immediate danger and moves toward food by Manhattan distance.

\section{Experiments}

\subsection{Experimental Setup}

We run 9 configurations (3 algorithms $\times$ 3 representations), each for 20,000 episodes across 5 random seeds. Performance is measured by the average score (food eaten per episode) over the final 1,000 episodes. We report mean and standard deviation across seeds. Learning curves are smoothed with a 100-episode moving average.

\subsection{Baseline Performance}

On the $20 \times 20$ grid (200 evaluation episodes), the random policy scores $0.15 \pm 0.38$ (max 2), dying primarily from wall collisions. The greedy heuristic scores $22.89 \pm 6.00$ (max 33), dying primarily from self-collision where it chases food greedily without planning around its own body. These establish lower and upper reference points.

\subsection{Analysis}

\subsubsection{Representation and Approximation Interaction.}

The results reveal a strong interaction between representation complexity and approximation method, summarized in Figure~\ref{fig:heatmap}. Tile coding achieves its best score on compact features (22.69), competitive with the greedy heuristic (22.89), but collapses to 3.21 on local and 1.32 on extended. Double DQN shows the opposite pattern: it achieves its highest score on extended features (29.05), exceeding the greedy heuristic by 27\%, and wins on every representation (24.26 compact, 22.51 local), including outperforming tile coding on compact (22.69) where tile coding is strongest. Linear FA is weak everywhere, peaking at 3.90 on extended.

This interaction aligns with theoretical expectations and is visible directly in the learning curves of Figure~\ref{fig:learning_curves} and the bar chart of Figure~\ref{fig:final_perf}. Tile coding builds generalization through spatial overlap in the tiled state space; this structure is well-defined in low dimensions but breaks down under hash collisions at high dimensionality. Neural networks learn generalization directly from data via gradient descent, making them more robust to representation choice when given sufficient training and a stable learning algorithm.

\subsubsection{Tile Coding vs.\ Hand-Crafted Features.}

Tile coding outperforms hand-crafted linear features by a factor of $28\times$ on the compact representation ($22.69$ vs.\ $0.80$). Tile coding's overlapping tilings create a much richer feature space than the 33-weight linear model: nearby states share most active tiles, enabling smooth value function generalization that binary features cannot provide.

However, tile coding's advantage disappears with higher-dimensional representations. With 109 or 126 input dimensions, the hash-based coder maps an astronomically large index space into 262,144 buckets, causing frequent collisions. Genuinely different states activate the same tile indices, corrupting the value function. Tile coding's scores drop monotonically with dimensionality: $22.69 \to 3.21 \to 1.32$ — a $17\times$ reduction from compact to extended. This is a known limitation of hash-based tile coding \cite{sherstov2005tilecoding} and highlights the necessity of dimensionality reduction or feature embedding before applying hash-based linear approximation in high-dimensional grid-world environments.

\subsubsection{Double DQN Stability Across Representations.}

One of the most striking results is Double DQN's uniformly low variance: $\pm 0.39$, $\pm 0.38$, and $\pm 0.32$ on compact, local, and extended representations respectively. This stability is not accidental. Experience replay breaks the temporal correlations that make sequential on-policy updates noisy, providing a more stationary regression problem. Double DQN's decoupled target reduces the overestimation feedback loop that destabilizes vanilla DQN. The hard-synchronized target network, copied every 100 episodes, prevents runaway value estimates by holding the regression target fixed within each update window.

The result is that Double DQN generalizes reliably even on the high-dimensional sparse local representation (22.51), where pure on-policy nonlinear function approximation typically struggles \cite{vanhasselt2018deadly}. This supports the view that for neural network function approximation in game environments, off-policy training infrastructure (replay + target network) is essential for consistent learning.

\subsubsection{Weight Analysis for Linear FA.}

The linear model's learned weights on the compact representation reveal interpretable structure. The dominant weights are the danger signals: the weight for ``danger straight'' in the STRAIGHT action block reaches approximately $-8$, and the analogous danger weights in the LEFT and RIGHT blocks follow the same pattern. The agent has learned a strong collision-avoidance policy.

However, food-direction weights are comparatively small and noisy ($< 2$ in magnitude). The linear model over 11 binary features can express avoidance but lacks expressiveness for nuanced food-seeking. Under a near-greedy policy ($\varepsilon \approx 0.01$), the agent enters deterministic loops because the compact representation maps many distinct game states to identical feature vectors, creating action cycles.

\subsubsection{Qualitative Behavior.}

The most striking contrast is between tile coding and linear FA on the compact representation. The tile coding agent navigates efficiently toward food, makes deliberate turns to avoid its own body, and consistently reaches scores above 20 (individual seeds reach a maximum of 63). The linear FA agent learns to avoid immediate collisions but exhibits repetitive looping, oscillating between two directions because the 11 binary features collapse distinct game states to the same vector.

The Double DQN agent on extended features displays the most sophisticated behavior: navigating toward food while maintaining spatial awareness of its body, occasionally taking longer detours to avoid self-trapping. Individual seeds reach maximum scores of 66, representing near-expert-level play. On the local representation, behavior is consistent across seeds ($\pm 0.38$), further confirming the stabilizing effect of experience replay.

\section{Conclusion}

We presented a systematic comparison of three function approximation methods across three state representations for learning to play Snake, using 20,000 training episodes and 5 random seeds. The key findings are: (1) Double DQN with experience replay achieves the highest and most consistent performance, reaching 29.05 on extended features and exceeding the greedy heuristic (22.89) by 27\%; (2) tile coding is an effective middle ground for compact, well-designed features (22.69) but collapses monotonically with dimensionality due to hash collisions; (3) linear FA learns danger avoidance but not food-seeking, remaining below 4.0 across all representations.

These results yield three practical insights. First, tile coding is a powerful and underappreciated middle ground: when features are low-dimensional and well-designed, it nearly matches neural network performance at a fraction of the computational cost. Second, representation design matters as much as algorithm selection: Double DQN scores 29.05 on extended but 22.51 on local — a 23\% variation driven purely by representation, not algorithm. Third, the stabilizing infrastructure of Double DQN and experience replay is critical for consistent neural network performance across diverse representations.

Limitations include the focus on a single domain (Snake), moderate hyperparameter tuning, and the absence of prioritized experience replay \cite{schaul2016prioritized}, which may further improve sample efficiency. Future work should investigate adaptive tile coding \cite{sherstov2005tilecoding} for high-dimensional inputs and continuous ray-casting representations to provide smoother gradients for function approximators.
